# Publishing And Route Ownership

## Route model

- `/` is the live monitor root. It is regenerated by the root exporter and should be treated as publisher-owned output.
- `/showcase/` is a hand-maintained route for durable project surfaces that should survive snapshot refreshes.
- `/tokenizers/` is the first showcase tool and remains hand-maintained inside the project repo.

## Current publisher boundary

The root exporter rewrites only `index.html` and `status.json` for this repo during each snapshot publish. That means:

- manual edits to the root homepage are transient unless the exporter itself changes
- subroutes such as `showcase/` and `tokenizers/` survive refreshes because the publisher does not overwrite them
- a persistent forward link from the root dashboard to a tool route requires a root-template or payload extension, not just a project-repo HTML patch

## Working rule

When a new browser experience should live on the site before the monitor exporter grows new extension points:

1. add it as a subroute in this repo
2. link it from `/showcase/`
3. document any missing root-homepage integration as an exporter issue instead of patching generated HTML by hand
