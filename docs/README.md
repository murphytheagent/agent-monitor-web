# agent-monitor-web Docs

- `dashboard-design-guide.md` - the current restrained-cyberpunk design contract: typography, accent policy, mobile hierarchy, and the exporter-owned boundary for `/` and `/roadmap/`
- `publishing.md` - publisher lifecycle (tmux session, restart procedure, config loading), route ownership, and the rule that generated-route changes belong in the exporter source
- `verification-workflow.md` - mandatory visual verification checklist for all website changes: full-page screenshots, live URL checks, mobile+desktop viewports, publisher restart for generated routes
- `site-shell.md` - the shared Murphy-branded header system for durable public routes, plus the current exporter-owned boundary
- `showcase-guidelines.md` - copy and hierarchy rules for both the `/showcase/` gallery and individual exhibit pages so future additions stay viewer-first instead of drifting back toward internal documentation
- `telemetry-showcase.md` - data sources, estimation rules, and manual refresh workflow for the `/showcase/signal-deck/` telemetry exhibit
